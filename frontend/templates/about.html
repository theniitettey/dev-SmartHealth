Smart Health Sync
Framework

A multi-class clinical decision-support system built as a final-year Computer Science dissertation project at the University of Ghana. It applies four supervised learning classifiers to predict six disease categories from 24 normalised blood-panel biomarkers — achieving 95.1% test accuracy with Random Forest as the production classifier.

Dataset Overview

The training corpus was assembled from multiple curated Kaggle datasets covering chronic and haematological conditions. After merging, cleaning, and deduplication, the final dataset contained 18,000+ records across six disease categories with 24 continuous biomarker features per record. Class imbalance was corrected using SMOTE (Synthetic Minority Oversampling Technique), and all features were normalised to a 0.0–1.0 scale before model training.

Section 3.5 — Dissertation

Exploratory Data Analysis

Prior to model training, a comprehensive EDA was conducted to understand feature distributions, inter-feature correlations, and class-wise biomarker behaviour. Key findings drove preprocessing decisions including outlier capping, SMOTE balancing, and feature retention strategy.

Distribution Analysis

Feature Distributions

Histograms were generated for all 24 continuous features. Glucose and HbA1c showed bimodal distributions — a clear separation between diabetic and non-diabetic populations. Haemoglobin exhibited left-skewed distribution in anaemic cases. Troponin and Creatinine were right-skewed with long tails, requiring log-scale inspection.

Key Findings
• Glucose bimodal at ~0.12 and ~0.85 (healthy vs. diabetic split)
• Haemoglobin strongly separates Anaemia from other classes
• Platelets showed near-normal distribution with anaemia outliers
• MCV and MCH co-vary strongly — correlated pair identified
Correlation Analysis

Correlation Heatmap

A Pearson correlation matrix was computed across all 24 features. Strongly correlated pairs (r > 0.85) were identified: MCV–MCH (r = 0.91), LDL–Total Cholesterol (r = 0.88), and RBC–Haematocrit (r = 0.87). All 24 features were retained to preserve predictive signal for the ensemble model.

Selection Rationale

Random Forest was selected as the production model for its superior accuracy (95.07%), lowest overfitting gap (CV mean 94.14% vs. test 95.07%), native support for probability estimation via predict_proba(), and built-in feature importance ranking. Its ensemble nature makes it inherently robust to noisy biomarker readings.

System Architecture

Classifier Descriptions

PRODUCTION Random Forest

An ensemble of decision trees trained using bagging (bootstrap aggregation). Each tree votes independently; the majority class wins. Chosen for its resistance to overfitting, built-in feature importance, and high accuracy on tabular clinical data.

VALID Support Vector Machine

Finds the optimal decision hyperplane maximising the margin between classes in high-dimensional space. Uses an RBF kernel to handle non-linearly separable biomarker patterns. Competitive accuracy (94.89%) but slower inference time.

VALID Decision Tree

Recursively splits the feature space using information gain. Highly interpretable — the full decision path can be visualised. Included to demonstrate a single-tree baseline and to contrast with the ensemble improvement Random Forest provides.

BASELINE Logistic Regression

A linear probabilistic classifier extended to multi-class via one-vs-rest (OvR) decomposition. Serves as the statistical baseline. Its 81.87% accuracy confirms the non-linear complexity of the disease classification task and validates the tree-based model choices.
